# Comparison Classification Guide

Consult this reference when producing comparison summaries (Step 5).

## Classify each finding as:

- **New**: Content not already present in `source`, `entity`, `concept`, or `wiki`. A fact, term, or relationship that would materially add to the repository.

- **Repeated**: Content already adequately covered in existing pages. Note where it appears; do not duplicate.

- **Conflicting**: Content that contradicts an existing page. Flag the contradiction explicitly — what the existing page says vs. what the new material says. Do not resolve unilaterally.

- **Pending review**: Content you cannot confidently classify. Flag and defer.

## Branching

If there is no prior content in higher layers: comparison is simpler — only flag internal conflicts within the new batch itself.
