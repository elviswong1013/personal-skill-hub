# ATS（申请人追踪系统）兼容性检查清单

## 背景

多数中大型企业使用 ATS 对简历做第一轮自动筛选。如果简历在 ATS 中解析失败或关键词密度不足，人工面试官根本看不到。

## A. 关键词与匹配

### 核心关键词分布

- 简历前三分之一（摘要 + 最近一段经历）是否包含 JD 中最核心的 3 到 5 个关键词
- 关键词是否以原文形式出现（ATS 通常做精确匹配，而非语义理解）
- 避免只在技能列表中出现一次——在项目描述中也要自然嵌入

### 关键词密度

- 每个核心关键词在简历中出现 2 到 4 次为宜
- 避免关键词堆砌（如 "Java, Java, Java"），会被部分 ATS 标记为作弊

### 技能拼写

- 技术术语是否使用标准拼写（如 "Node.js" 而非 "NodeJS"、"React" 而非 "reactjs"）
- 同一技能是否前后拼写统一

## B. 格式兼容性

### Section 标题

- 使用 ATS 友好的标准标题：
  - "Work Experience" / "工作经历"
  - "Education" / "教育经历"
  - "Skills" / "技能"
  - "Projects" / "项目经历"
- 避免创意标题，如 "Where I've Been"、"What I Can Do"

### 排版陷阱

- 不要使用表格、文本框、图片来承载关键信息
- 不要使用页眉/页脚放联系方式（部分 ATS 会忽略页眉页脚区域）
- 不要使用多栏布局
- 不要使用特殊符号/图标作为 bullet（用标准 "-" 或 "·"）

### 文件格式

- 优先使用 .docx 或纯文本 .pdf
- 避免图片型 PDF（从 LaTeX/设计工具导出后务必验证文本可选中、可复制）
- 文件名建议格式：`姓名_岗位_简历.pdf`，避免中文文件名在部分系统中乱码

## C. 联系方式可解析性

- 手机号、邮箱是否使用纯文本（非图片/非特殊编码）
- 邮箱是否为常见域名（@qq.com、@gmail.com、@163.com 均可）
- LinkedIn/GitHub 链接是否完整且可点击

## D. 自查方法

投递前快速验证：

1. 将简历全文复制到记事本，检查是否有乱码、缺字、顺序错乱
2. 用 Ctrl+F 搜索 JD 中的核心关键词，确认在简历中出现且分布在多个位置
3. 确认从 PDF 中能完整复制所有文字（不可复制的 PDF 对 ATS 基本不可读）
4. 将简历保存为 .txt，确认所有关键信息（姓名、联系方式、公司名、时间段）完整保留
