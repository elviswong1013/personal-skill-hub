# 一页简历压缩规则

## 目标

一页简历不是删字游戏，而是把最能提高面试转化率的信息压到一页内。核心目标是：

- 让招聘方在 30 秒内看懂你是谁
- 快速看到你最相关的成果
- 减少不必要的背景噪音

## 压缩原则

### 0. 何时不建议压缩到一页

以下情况不要强行压缩到一页：

- 15 年以上经验的高阶/管理岗位（总监及以上），需要展示职业成长路径和多段关键经历
- 学术界或研究型岗位，需要完整的发表列表、专利清单或研究项目
- 多段高度相关且不可删除的经历（如 3 段以上核心业务从 0 到 1 的经历，每段都需要完整上下文）
- 投递咨询、金融等对详细履历有格式要求的行业

以上情况可压缩到两页，但依然遵循相同的压缩原则：删除弱相关内容和低价值 bullet。

### 1. 相关性优先

优先保留：

- 最近 5 到 8 年内的经历
- 与目标 JD 高度相关的岗位、项目和技术
- 最能体现职级、复杂度和影响力的成果

优先删除或弱化：

- 明显弱相关的早期经历
- 没有结果的职责描述
- 重复的技能清单
- 空洞自评和长篇个人评价

### 2. 每段经历只留高价值 bullet

通常每段经历保留 2 到 4 条 bullet，优先级从高到低：

1. 最能证明岗位匹配度的成果
2. 最能证明技术深度或业务复杂度的成果
3. 最能证明影响力、 ownership 或跨团队协作的成果
4. 其他支持性信息

### 3. 每条 bullet 都要高密度

优先满足这个结构：

```text
动作 + 产物 + 结果
```

更强的版本是：

```text
问题/目标 + 动作 + 产物 + 结果
```

### 4. 技能清单只保留筛选价值

不要把技能列表写成百科。只保留：

- JD 中会被检索的核心关键词
- 能被项目经历证明的能力
- 当前方向真正相关的技术

## 推荐版式

一页版通常保留以下模块：

1. 标题和联系方式
2. 1 到 2 行职业摘要
3. 核心技能
4. 工作经历
5. 重点项目或教育经历

如果工作经历已经足够强，项目和教育可以极度压缩。

## 输出模板

```markdown
# 姓名 | 目标岗位

## 职业摘要
[1 到 2 行，写方向、年限、最强价值]

## 核心技能
[按岗位筛选价值保留 6 到 12 个关键词]

## 工作经历
### 公司 / 岗位 / 时间
- [高密度成果 bullet]
- [高密度成果 bullet]
- [高密度成果 bullet]

### 公司 / 岗位 / 时间
- [高密度成果 bullet]
- [高密度成果 bullet]

## 项目 / 教育
[只保留最相关内容]
```

## 判断是否压得够好的标准

一页版至少要做到：

- 第一屏就能看出目标岗位
- 最近经历比旧经历更突出
- 至少有 3 条以上能体现结果的 bullet
- 技能、经历、目标岗位之间能互相印证
